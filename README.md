# 盲盒自动售货机补货与收益平台

## 项目简介

本项目是一个盲盒自动售货机运营管理平台，采用前后端分离架构，为运营公司、供应商、商场、补货员和客服五种角色提供全方位的管理功能。

**核心特色**：前台展示概率、真实格口库存、补货记录和收益分账完全对齐，确保用户看到的隐藏款概率与机器里的实际库存始终一致。

## 原始需求

> 请实现盲盒自动售货机补货与收益平台，Vue3 后台给运营公司、供应商、商场、补货员和客服使用，Spring Boot 保存机器点位、IP 系列、格口库存、抽盒概率、支付订单、出货电机、退款、补货路线、商场扣点和收益分账。运营公司选择系列和投放点；供应商发货并声明隐藏款规则；补货员开柜扫码补货、清点剩余和处理卡货；商场查看销售与停机时长；客服处理重复扣款、未出货和概率质疑。这个产品要把前台展示概率、真实格口库存、补货记录和收益分账对齐，不能让用户看到的隐藏款概率和机器里的实际库存脱节。

## 启动方式

### 前置要求

- Docker & Docker Compose（推荐方式）
- 或者：JDK 17+、Node.js 18+、MySQL 8.0+、Maven 3.8+

### 方式一：Docker Compose 一键启动（推荐）

#### 1. 启动服务

在项目根目录执行：

```bash
docker compose up --build
```

如需后台运行：

```bash
docker compose up --build -d
```

#### 2. 停止服务

```bash
docker compose down
```

#### 3. 访问地址

- 前端管理后台：http://localhost
- 后端 API 接口：http://localhost:8080/api
- MySQL 数据库：localhost:3306

#### 4. 数据库信息

- 数据库名：blindbox
- 用户名：blindbox
- 密码：blindbox123

> 注意：首次启动时 MySQL 会自动执行初始化脚本，创建表结构和初始数据。

### 方式二：本地开发启动

#### 后端启动

1. 确保 MySQL 服务已启动，创建数据库：

```sql
CREATE DATABASE blindbox DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

2. 执行初始化脚本：`backend/sql/init.sql`

3. 根据本地环境修改 `backend/src/main/resources/application.yml` 中的数据库连接信息。

4. 启动后端：

```bash
cd backend
mvn clean install -DskipTests
mvn spring-boot:run
```

后端访问地址：http://localhost:8080/api

#### 前端启动

```bash
cd frontend
npm install
npm run dev
```

前端访问地址：http://localhost:3000

## 核心业务说明

### 概率与库存对齐机制

本系统的核心特色是**基于实际库存计算真实概率**，确保前台展示概率与实际库存完全一致：

1. 每个机器的格口库存是真实的物理库存
2. 展示概率 = 该SKU库存格口数 / 总库存格口数
3. 抽盒时，从实际有货的格口中随机抽取（权重即库存数量）
4. 抽中后从对应格口中扣减库存，概率自动更新
5. 每次抽盒后自动进行概率审计，记录审计日志
6. 补货后也会进行概率审计检查

### 收益分账规则

- **商场扣点**：按商场设置的佣金比例从订单总额中扣除
- **平台分成**：剩余金额的 10%
- **供应商分成**：剩余金额的 60%
- **运营公司分成**：剩余部分

### 概率审计

- 补货后自动审计
- 每次抽盒后自动审计
- 客服可手动审计
- 审计日志记录实际库存与展示概率的偏差
- 不一致时会触发预警标志

## API 接口概览

### 认证模块
- `POST /api/auth/login` - 登录
- `GET /api/auth/profile` - 获取当前用户信息

### 抽盒接口（无需登录，模拟C端）
- `GET /api/draw/machine/{machineCode}` - 获取机器详情（含库存和实际概率）
- `POST /api/draw/do` - 抽盒

- 收益查看
